# Python Coded Agent Invocation Recipe

## What this page is

End-to-end recipe for invoking a UiPath-deployed agent (Python coded or low-code) from a Maestro BPMN process: how to declare the entry input, mirror it into mutable state, call the deployed agent, capture the JSON output, and drive a downstream decision from one field of that output. This recipe is targeted at the common gap where a model has the agent already deployed in a folder but no documented BPMN path to invoke it.

## Supported paths

Two implementation paths are documented. The recommended path is the one a real working solution uses today; the direct path is documented based on user feedback and is not yet verified end-to-end.

### Recommended (verified): RPA-wrapper indirection

Wrap the deployed agent in a thin RPA (XAML) project — a single `RunJob` activity invoking the agent with `DataMappingPocoActivity` translating input/output JSON to and from the agent's strongly-typed bundle classes — and invoke that RPA process from BPMN via `Orchestrator.StartJob`. This is the contract the BPMN side already validates against and the path a working solution uses today.

Tradeoffs:

- Doubles per-agent surface area: each deployed agent needs a sibling XAML wrapper project in the solution.
- Uses the verified `Orchestrator.StartJob` BPMN contract (see [rpa-job.md](rpa-job.md)).
- Produces strongly-typed I/O via Studio Web's `DataMappingPocoActivity` and auto-generated input/output bundle classes.

Known product gap: the strongly-typed input/output bundle classes are currently auto-generated by Studio Web and not reproducible from a CLI-only workflow. Authoring the XAML wrapper itself is an `uipath-rpa` skill task; this skill does not own that XAML.

### Direct invocation (draft - verification-required)

`Orchestrator.StartAgentJob` against a deployed agent, with a pair-binding shape in `uipath:bindings`. User feedback reports that `uip solution pack` validates the binding-pair shape (two `uipath:binding` rows with `propertyAttribute="name"` and `propertyAttribute="folderPath"` pointing at the same `resourceKey`, with `resourceSubType="Agent"`); this has not been verified at runtime in this skill's fixtures. Treat the direct-invocation path as draft until an operator has packed and run it.

Root-level bindings block:

```xml
<uipath:bindings version="v1">
  <uipath:binding id="Binding_Agent_Name" name="Synthetic Agent" type="agent"
    elementId="Task_StartAgentJob" resource="SyntheticAgent"
    resourceSubType="Agent" resourceKey="synthetic-agent-key"
    propertyAttribute="name" />
  <uipath:binding id="Binding_Agent_FolderPath" name="Synthetic Agent Folder" type="agent"
    elementId="Task_StartAgentJob" resource="SyntheticAgent"
    resourceSubType="Agent" resourceKey="synthetic-agent-key"
    propertyAttribute="folderPath" />
</uipath:bindings>
```

The service task references both bindings in `uipath:context`. See [`../../../shared/wrapper-shells.md`](../../../shared/wrapper-shells.md) for the matching wrapper shell.

## End-to-end example (recommended path)

The fragment below uses the recommended RPA-wrapper path. Synthetic IDs and resource keys only. For the full runnable BPMN with diagram metadata, see `../../../../fixtures/validation/agent-invocation-e2e/`.

Entry-point input and the mirror variable on the process:

```xml
<uipath:variables version="v1">
  <uipath:input id="Var_RequestId" name="requestId" type="string" elementId="Start_Request" />
  <uipath:inputOutput id="Var_RequestId_Mirror" name="requestIdMirror" type="string" />
  <uipath:inputOutput id="Var_JobStatus" name="jobStatus" type="string" />
  <uipath:inputOutput id="Var_AgentResult" name="agentResult" type="json" />
</uipath:variables>
<uipath:bindings version="v1">
  <uipath:binding id="Binding_AgentWrapperProcess" name="Agent Wrapper Process"
    type="process" elementId="Task_InvokeAgent" resource="SyntheticAgentWrapper"
    resourceSubType="process" resourceKey="synthetic-agent-wrapper-process" />
</uipath:bindings>
```

Start event mapping copies the read-only entry input into a mutable mirror variable. See [`../../../shared/variables-bindings-expressions.md`](../../../shared/variables-bindings-expressions.md) for why this re-projection is required.

```xml
<bpmn:startEvent id="Start_Request" name="Start">
  <bpmn:extensionElements>
    <uipath:entryPointId value="Entry_AgentInvocation" />
    <uipath:mapping version="v1">
      <uipath:type value="BPMN.Variables" version="v1" />
      <uipath:context />
      <uipath:output name="requestIdMirror" type="string" var="Var_RequestId_Mirror" source="=vars.Var_RequestId" />
    </uipath:mapping>
  </bpmn:extensionElements>
  <bpmn:outgoing>Flow_Start_To_Invoke</bpmn:outgoing>
</bpmn:startEvent>
```

Service task invokes the XAML wrapper via `Orchestrator.StartJob`, passing the mirror variable as JSON and capturing both status and the agent's structured result:

```xml
<bpmn:serviceTask id="Task_InvokeAgent" name="Invoke Agent">
  <bpmn:extensionElements>
    <uipath:activity version="v1">
      <uipath:type value="Orchestrator.StartJob" version="v1" />
      <uipath:context>
        <uipath:input name="process" type="resource" value="=bindings.Binding_AgentWrapperProcess" />
      </uipath:context>
      <uipath:input name="JobArguments" type="json" target="bodyField"><![CDATA[{"requestId":"=vars.Var_RequestId_Mirror"}]]></uipath:input>
      <uipath:output name="JobState" type="string" var="Var_JobStatus" source="state" />
      <uipath:output name="JobResult" type="json" var="Var_AgentResult" source="result" />
    </uipath:activity>
  </bpmn:extensionElements>
  <bpmn:incoming>Flow_Start_To_Invoke</bpmn:incoming>
  <bpmn:outgoing>Flow_Invoke_To_Gateway</bpmn:outgoing>
</bpmn:serviceTask>
```

A downstream exclusive gateway reads one field of the agent's JSON output to choose a branch:

```xml
<bpmn:exclusiveGateway id="Gateway_Outcome" name="Outcome" default="Flow_Gateway_To_Rejected">
  <bpmn:incoming>Flow_Invoke_To_Gateway</bpmn:incoming>
  <bpmn:outgoing>Flow_Gateway_To_Approved</bpmn:outgoing>
  <bpmn:outgoing>Flow_Gateway_To_Rejected</bpmn:outgoing>
</bpmn:exclusiveGateway>
<bpmn:sequenceFlow id="Flow_Gateway_To_Approved" sourceRef="Gateway_Outcome" targetRef="End_Approved">
  <bpmn:conditionExpression xsi:type="bpmn:tFormalExpression" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">=vars.Var_AgentResult.outcome == "approved"</bpmn:conditionExpression>
</bpmn:sequenceFlow>
```

## The XAML wrapper template

The XAML wrapper itself is not part of this skill. At the level of detail a BPMN model author needs:

- A single XAML workflow whose root activity is `RunJob`, configured to invoke the deployed coded agent.
- A `DataMappingPocoActivity` translating the incoming JSON `JobArguments` payload into the agent's strongly-typed input bundle class, and the agent's strongly-typed output bundle class back into the JSON returned to BPMN.
- The strongly-typed input/output bundle classes are auto-generated by Studio Web at design time; reproducing them outside Studio Web is not currently supported.

Authoring the XAML wrapper is a sibling-skill task. See the `uipath-rpa` skill for XAML authoring and the BPMN side at [`../plugins/rpa-jobs/`](../plugins/rpa-jobs/) for the `Orchestrator.StartJob` contract this recipe relies on. This skill does not import or reference cross-skill content directly.

## Why not `A2A.AgentExecution`?

`A2A.AgentExecution` is the cross-tenant / external A2A primitive. Studio Web's visual editor disables the Action dropdown for this wrapper and surfaces `agentUrl`, `skillId`, and `authToken` fields that do not apply to a folder-deployed UiPath agent. Use `A2A.AgentExecution` only when calling an external A2A endpoint, not when invoking an agent the same tenant has deployed via Orchestrator.

## Validation checklist

Before claiming this recipe runs end-to-end:

- Mirror variable (`uipath:inputOutput`) exists and is the target of a `BPMN.Variables` mapping on the start event that consumes the read-only `uipath:input`.
- If using the draft direct path, the `uipath:bindings` block contains both `propertyAttribute="name"` and `propertyAttribute="folderPath"` rows for the agent, with matching `resourceKey` and `resourceSubType="Agent"`.
- Output mapping `var` targets a declared `uipath:inputOutput` or `uipath:output` (read-only `uipath:input` will be rejected).
- `uip solution pack <path>` passes. Local `uip maestro bpmn validate` is necessary but not sufficient; the deploy-time `solution pack` validator is the authority on the binding-pair shape.
- For the draft direct path, an operator has packed and run the BPMN against a real tenant before relying on the runtime contract.
