# Mail Activities Playbooks

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Send Outlook Mail Message Failures | Medium | `Send Outlook Mail Message` (`SendOutlookMail`, `UiPath.Mail.Outlook.Activities`) fails across three surfaces: COM cast / library-not-registered (`Unable to cast COM object` / `Library not registered` — Outlook not installed/registered, process-vs-Outlook bitness mismatch, corrupted Office registry, or orphaned `OUTLOOK.EXE`); activity times out or hangs (hidden "a program is trying to send an email" security prompt, Work Offline mode, or slow profile load); and `Object reference not set to an instance of an object` from an uninitialized `To`/`Subject`/`Body` or empty attachment path. Includes the `Send SMTP Mail Message` / modern Graph fallback for unattended runs. | [send-outlook-mail-failures.md](./playbooks/send-outlook-mail-failures.md) |
| Get Outlook Mail Messages Failures | Medium | `Get Outlook Mail Messages` (`GetOutlookMailMessages`, `UiPath.Mail.Outlook.Activities`) fails across five surfaces: folder not resolved (`The specified folder does not exist` — nested-path backslash syntax, shared mailbox needing the `Account` field, or a localized folder name); timeout on a large folder (`The operation has timed out` — raise `TimeoutMS`, lower `Top`, enable `OnlyUnreadMessages`); Outlook not running / COM session or privilege mismatch (`The Outlook application is not running` / freeze on unattended RDP, or a UiPath-vs-Outlook elevation mismatch); malformed DASL/Jet `Filter` (unquoted value yields an error or zero results); and Cached Exchange Mode desync (no error, but recent/old mail visible in the client is missing). Shares the COM surface with Send Outlook Mail; cross-references the modern Graph o365 reads for unattended runs. | [get-outlook-mail-failures.md](./playbooks/get-outlook-mail-failures.md) |
| Move Outlook Mail Message Failures | Medium | `Move Outlook Mail Message` (`MoveOutlookMessage`, `UiPath.Mail.Outlook.Activities`) fails across four surfaces: COM session loss (`The operation failed` / `The specified folder does not exist` — intermittent or 100% on unattended robots when Outlook is closed/asleep/detached; fix with a live foreground Outlook, Startup apps, and a Retry Scope); destination folder path / `Account` mismatch (deterministic `The folder does not exist` — use `Inbox\Archive` not `Archive`, set `Account` for shared mailboxes); modern-vs-classic type mismatch (`Cannot convert type 'String' to 'IResource' / 'Office365Message'` after a dependency update — map the resource with the `+` picker, or use the classic activity with string/`MailMessage` inputs); and the New Outlook regression (a system update flips the host to New Outlook, which removes the Desktop COM API — toggle back to Classic Outlook or migrate to the Graph o365 Move Email). | [move-outlook-mail-failures.md](./playbooks/move-outlook-mail-failures.md) |
| Delete Outlook Mail Message Failures | Medium | `Delete Outlook Mail Message` (`DeleteOutlookMailMessage`, `UiPath.Mail.Outlook.Activities`) fails across five surfaces: stale message reference (`The operation failed. An object could not be found.` — the item was moved/deleted between Get and Delete; fetch-then-delete promptly + Try Catch); collection modified in loop (`Collection was modified; enumeration operation may not execute.` — deleting inside a `For Each` over the live message list; iterate a `.ToList()` copy / backwards, or Move-then-delete); New Outlook regression (removes the Desktop COM API — revert to Classic or migrate to Graph); COM session blocked / privilege (`The operation has timed out` / freeze from a modal programmatic-access dialog or a UiPath-vs-Outlook elevation mismatch; match privilege, fix Trust Center); and mailbox permission / access denied (Robot account lacks Full Access on a shared mailbox; set `Account` and grant Full Access via Exchange). | [delete-outlook-mail-failures.md](./playbooks/delete-outlook-mail-failures.md) |
| Save Mail Failures | High | `Save Mail` (`SaveMail`, `UiPath.Mail.Activities`) throws `System.ArgumentException` at the node when `FilePath` is invalid — illegal path characters (often a raw subject/sender spliced into the filename), empty/unresolved, or a malformed/reserved name; `System.IO`/`FileInfo` reject it before writing the `.eml`. Distinct from path-too-long (IOException). | [save-mail-failures.md](./playbooks/save-mail-failures.md) |
| Save Mail Attachments Failures | Medium | `Save Attachments` / `Save Mail Attachments` (`SaveMailAttachments`, `UiPath.Mail.Activities`) throws `System.IO.IOException` / `PathTooLongException` writing attachments: target file already exists (`Overwrite existing files` off), file locked/in-use, destination unavailable/full/access-denied, or path too long; download-stage errors wrap as `Error downloading attachments`. Illegal filename-filter characters throw `ArgumentException`. | [save-mail-attachments-failures.md](./playbooks/save-mail-attachments-failures.md) |
| Create HTML Content Failures | High | `Create HTML Content` (`Business.CreateHtmlContent`, `UiPath.Mail.Activities`) throws `System.ArgumentException` validating an embedded-resource/HTML file path: `Directory "<path>" doesn't exist.`, `There are invalid characters in the file path.`, or `File "<path>" does not exist.` — a missing/dev-only file, a relative path resolved against the Robot's working dir, or an unsanitized dynamic path segment. | [create-html-content-failures.md](./playbooks/create-html-content-failures.md) |
| Outlook Application Card Failures | Medium | `Use Outlook 365` / `Outlook Application Card` (`Business.OutlookApplicationCard`, `UiPath.Mail.Activities`) fails opening the Outlook COM session: `System.Runtime.InteropServices.COMException` (Outlook not installed/registered, bitness mismatch, corrupt type library, orphaned `OUTLOOK.EXE`, or New Outlook removing the COM API) or `System.SystemException` — `Cannot open Outlook. Outlook is already open in another session…` (session conflict) / `The account <name> could not be found.`. Prefer Graph o365 for unattended. | [outlook-application-card-failures.md](./playbooks/outlook-application-card-failures.md) |
| For Each Email Failures | Medium | `For Each Email` (`Business.ForEachEmailX`, `UiPath.Mail.Activities`) faults iterating a lazy provider collection: `System.TimeoutException` (per-item provider calls / attachment downloads on a slow or large mailbox) or `System.SystemException` (provider session dropped mid-loop, `…underlying email provider…could not be determined` on an attachment download, or a `Filter` applied to a manually-built list — `…created manually…remove the filters`). | [for-each-emailx-failures.md](./playbooks/for-each-emailx-failures.md) |
| Send Mail (SendMailX) Failures | Medium | `Send Mail` (`Business.SendMailX`, `UiPath.Mail.Activities`) propagates provider exceptions raw (only innermost `HResult 0x8000FFFF` wraps to `Cannot connect to Outlook…`): `System.ObjectDisposedException` (connection used outside/after its scope), `Microsoft.Graph.ServiceException` (Graph `429` throttling / `401`-`403` auth-permission / `400` bad recipient / `404` mailbox), or `Google.GoogleApiException` (Gmail auth-scope / quota / invalid recipient). `The Mail service is null.` = unresolved connection. | [send-mailx-failures.md](./playbooks/send-mailx-failures.md) |
| Get Exchange Mail Messages Failures | Medium | `Get Exchange Mail Messages` (`GetExchangeMailMessages`, `UiPath.Mail.Exchange.Activities`) over EWS: `ServiceVersionException` (`…only valid for Exchange Server version Exchange2010 or a later version` — raise `ExchangeVersion`), `System.AggregateException` (per-message bind failures — expand inner), `ServiceResponseException` (folder/item not found, access denied), or `UiPath.Mail.ExchangeException` (`Authentication failed for user <user>.`; folder `Mail Folder does not exist for specified client.`). | [get-exchange-mail-messages-failures.md](./playbooks/get-exchange-mail-messages-failures.md) |
| Send Exchange Mail Failures | Medium | `Send Exchange Mail Message` (`SendExchangeMail`, `UiPath.Mail.Exchange.Activities`) over EWS clusters into connection/auth (`System.UriFormatException` bad EWS URL, `MsalClientException` OAuth, `ServiceRequestException` transport/autodiscover, `ExchangeException` `Authentication failed for user <user>.`), inputs (`NullReferenceException`/`ArgumentNullException`/`ArgumentException`/`KeyNotFoundException` — missing/invalid To/From/account/lookup key), and attachments (`System.IO.FileNotFoundException` — path missing on the Robot host). | [send-exchange-mail-failures.md](./playbooks/send-exchange-mail-failures.md) |
| Get IMAP Mail Messages Failures | Medium | `Get IMAP Mail Messages` (`GetIMAPMailMessages`, `UiPath.Mail.IMAP.Activities`, MailKit) — connect failures wrap in `Cannot connect to the (IMAP) Mail Service…` with the **inner** type as discriminator: `SslHandshakeException` (port/`SecureConnection` mismatch, untrusted cert, CRL), `System.TimeoutException` (host/port unreachable), `AuthenticationException` (wrong creds / app-password / OAuth / IMAP disabled), or `ImapProtocolException` (malformed/dropped response). Folder: `Mail Folder does not exist…`. | [get-imap-mail-messages-failures.md](./playbooks/get-imap-mail-messages-failures.md) |
| Move IMAP Mail Message To Folder Failures | Medium | `Move IMAP Mail Message To Folder` (`MoveIMAPMailMessageToFolder`, `UiPath.Mail.IMAP.Activities`, MailKit): `MailKit.Net.Imap.ImapCommandException` (destination folder doesn't exist — the activity creates nothing; wrong case / hierarchy separator; insufficient rights) or `System.IO.IOException` (session dropped mid-move — idle gap between fetch and move, network blip). | [move-imap-mail-message-to-folder-failures.md](./playbooks/move-imap-mail-message-to-folder-failures.md) |
| Send SMTP Mail Message Failures | Medium | `Send SMTP Mail Message` (`SendMail`, `UiPath.Mail.SMTP.Activities`, MailKit): inputs/format (`System.FormatException` malformed address, `ArgumentException`/`ArgumentNullException`/`NullReferenceException`/`KeyNotFoundException`), TLS (`SslHandshakeException` — `StartTls`/587 vs `SslOnConnect`/465, cert), or `SmtpCommandException` by SMTP status (`5.7.x` relay/auth, `550` recipient, size). Connect failures wrap in `Cannot connect to the (SMTP) Mail Service…`. | [send-smtp-mail-failures.md](./playbooks/send-smtp-mail-failures.md) |
