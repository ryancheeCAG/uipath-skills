# Google Workspace Activities Presentation Rules

- **Activities** — use the display name in prose (e.g., "Get Newest Email", "Add Sheet", "Create Folder"). Include the class name in parentheses on first mention so an agent can match a stack trace (e.g., "Get Newest Email (`GetNewestEmailConnections`)"). Don't lead with the fully qualified class name.
- **Connections** — use the connection name as configured in Integration Service (e.g., "gsuiteactivities"), not internal IDs. When the underlying account matters for the diagnostic, also state the authenticated Google account email.
- **Drive items (files / folders)** — refer to a Drive file or folder by its title with parent context (e.g., "folder 'Reports' under '/Projects/2024'"), not by raw Drive ID. Include the ID or URL only when the diagnostic requires it — for example, when the playbook is the 404 case and the ID itself is what failed to resolve.
- **Spreadsheets** — refer to a spreadsheet by its title (e.g., "spreadsheet 'Q1 Forecast'"), not by spreadsheet ID. Show the URL/ID only when the user needs to open it or the ID itself is the subject of the error.
- **Sheets (tabs inside a spreadsheet)** — refer to a sheet by its title (e.g., "sheet 'Data'"), not by `sheetId`. Name comparison for `Add Sheet` is case-insensitive — when reporting a conflict, show both the configured name and the existing sheet's title verbatim.
- **Gmail folders / labels** — use the user-visible label name (e.g., "Inbox", "Sent", "label 'Customers/VIP'"), not the internal Gmail label ID (`Label_123`, `INBOX`, `SENT`). When reporting filter scope, include the configured `Folder` value as the user typed it.
- **`ConflictResolution` values** — refer to conflict-behavior values by the activity-facing enum display names. When recommending a value to the user, also state in plain language what it does (e.g., "set `ConflictResolution = Rename` so the new item is auto-renamed").
- **Local file paths (upload sources)** — show the full local path as configured on the activity (e.g., `C:\Reports\invoice.pdf`), not just the file name. Local paths and Drive paths look similar — clarify which one you mean.
- **Shared drives** — distinguish "My Drive" (the user's personal Drive) from a shared drive, and refer to a shared drive by its display name (e.g., "shared drive 'Finance Ops'"), not by ID. Storage and file-count limits behave differently between the two.
