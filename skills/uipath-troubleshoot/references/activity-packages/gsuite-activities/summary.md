# Google Workspace Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for Google Workspace Activities investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Get Newest Email — No Matching Email Found | High | Gmail mailbox contains zero messages matching the configured filter at the moment the activity runs (`GmailException: No email matching the search criteria has been found`) | [get-newest-email-not-found.md](./playbooks/get-newest-email-not-found.md) |
| Drive — Name Conflict in Specified Folder | High | Drive activity (CreateFolder, CopyFile, MoveFile, Rename, Upload, CreateSpreadsheet, CreateDocument) throws `GSuiteException: Multiple items with the name <name> found in the specified folder.` because the configured `ConflictResolution` cannot resolve a duplicate name | [drive-multiple-items-name-conflict.md](./playbooks/drive-multiple-items-name-conflict.md) |
| Add Sheet — Sheet Name Already Exists | High | Target spreadsheet already contains a sheet (tab) with the configured `SheetName` (case-insensitive) and `ConflictResolution = Fail` (`GSuiteException: A sheet with the same name already exists.`) | [add-sheet-name-conflict.md](./playbooks/add-sheet-name-conflict.md) |
| Drive — File or Folder Not Found (404) | High | Drive activity fails to resolve the configured ID/URL/path because the resource was deleted, trashed, or is not accessible to the connection (`File not found: <id>. [404]` / `HttpStatusCode is NotFound` / `The resource was not found.`) | [drive-file-not-found.md](./playbooks/drive-file-not-found.md) |
| Upload — Drive Storage Quota or Shared Drive File Limit Exceeded | High | Upload activity (Drive `UploadFilesConnections`, legacy `UploadFile`, or Gmail send/reply/forward with attachments) fails because the user's Drive storage is full (`storageQuotaExceeded`) or the destination shared drive has hit the 500K item limit (`Upload failed after <N> bytes. ...`) | [upload-storage-quota-exceeded.md](./playbooks/upload-storage-quota-exceeded.md) |
| Sheets — 10 Million Cell Limit Exceeded | High | Sheets write/expand activity (Write Range/Cell/Row/Column, AutoFill, CopyPaste, Add Sheet, plus legacy AppendRow, CopySheet, BatchUpdateValuesScope, Add modes of AddDelete*) fails because the operation would push the spreadsheet past Google's 10M cell cap (`HttpStatusCode is BadRequest. This action would increase the number of cells in the workbook above the limit of 10000000 cells.`) | [sheets-cell-limit-exceeded.md](./playbooks/sheets-cell-limit-exceeded.md) |
| Sheets — Invalid Range / Unable to Parse Range | Medium | Sheets read/write activity passes a malformed A1-notation range to the Sheets API (`HttpStatusCode is BadRequest. Invalid data[0]: Unable to parse range: <SheetName>!<CellRef>`). Common causes: 1-indexing mistake (row 0), empty/null range, unquoted sheet names with spaces, stale sheet references, off-by-one in dynamic construction | [sheets-invalid-range.md](./playbooks/sheets-invalid-range.md) |
