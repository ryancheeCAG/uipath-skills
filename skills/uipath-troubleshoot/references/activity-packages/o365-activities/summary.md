# Microsoft Office 365 Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for Microsoft Office 365 Activities investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Get Newest Email — No Email Matched the Filter Criteria | High | Mailbox contains zero messages matching the configured filter at the moment the activity runs (`Office365Exception: No email matching the filter criteria, received in the last 1 hour has been found...`). | [get-newest-email-no-match.md](./playbooks/get-newest-email-no-match.md) |
| Item Name Already Exists | High | Destination already contains an item with the target name and `ConflictBehavior = Fail` (`Office365Exception: The specified item name already exists.`). Raised by Files (`CreateFolderConnections`, `CopyItemConnections`, `MoveItemConnections`, `RenameItemConnections`, `UploadFilesConnections`), Excel (`CreateWorkbookConnections`, `AddSheetConnections`, `RenameSheetConnections`), and legacy activities equivalents. Conflict is on the OneDrive/SharePoint/Excel side. | [item-name-already-exists.md](./playbooks/item-name-already-exists.md) |
| Download — Multiple Items Name Conflict | High | Download activity (`DownloadEmailConnections`, `DownloadFileConnections`, `DownloadEmailAttachments`) throws `Office365Exception: Multiple items with the name <name> found in the specified folder.` because the **local** destination folder already contains a file with that name and `ConflictResolution = Fail`, or two attachments on the same email resolve to the same local filename. | [download-multiple-items-name-conflict.md](./playbooks/download-multiple-items-name-conflict.md) |
| Mail — Folder Not Found | High | Mail activity (`MoveEmailConnections`, `GetNewestEmail`, `GetEmailListConnections`, `ForEachEmailConnections`, `WaitForEmailReceived`, `NewEmailReceived` trigger, legacy `MoveMail`) cannot resolve the configured `MailFolder`. Surfaces as `Office365Exception: The resource could not be found.`, `Folder named '<name>' could not be found on this account.`, or `Cannot find item configured with connection <connection> at path <folder>.`. | [mail-folder-not-found.md](./playbooks/mail-folder-not-found.md) |
| Drive — File or Folder Not Found | High | OneDrive/SharePoint Drive activity (`GetFileFolderConnections`, `GetFileFolderMetadataConnections`, `GetFileListConnections`, `DownloadFileConnections`, `MoveItemConnections`, `CopyItemConnections`, `RenameItemConnections`, `DeleteItemConnections`, `ShareItemConnections`, `UpdateFileFolderMetadataConnections`, `CreateFolderConnections`, `UploadFilesConnections`, `ForEachFileFolderConnections`, `CheckinCheckoutFileConnections`, `AssignSensitivityLabelConnections`, `GetSensitivityLabelsConnections`, persistence `WaitForFile*`, legacy `FindFilesAndFolders`, `ForEachFileFolder`, `CopyItem`, `UploadFile`, `DownloadFile`, `ExportAsPdf`, `GetItem`, `MoveItem`, `DeleteItem`, `ShareItem`, `CreateFolder`) cannot resolve the configured item. Surfaces as `Office365Exception: The resource could not be found.`, the wrapping form `Cannot find item configured with connection <connection> at path <path>.`, or on legacy activities a raw `Microsoft.Graph.ServiceException` with `Code: itemNotFound` and the same sentence. | [drive-item-not-found.md](./playbooks/drive-item-not-found.md) |
| Insufficient Graph Scope / Access Denied (403) | High | Authenticated caller lacks the Microsoft Graph permission for the operation. Surfaces as `The caller doesn't have permission to perform the action.`, `Access restricted to the item's owner.`, or a raw `Insufficient privileges to complete the operation.` Common on write/share ops (`SendMailConnections` → Mail.Send; `ShareItemConnections`, `UploadFilesConnections`, `MoveItemConnections` → Files.ReadWrite.All / Sites.ReadWrite.All). Distinct from 404 not-found — Graph may return 404 instead of 403 for unauthorized cross-mailbox/shared access. | [insufficient-graph-scope.md](./playbooks/insufficient-graph-scope.md) |
| Throttling / Rate Limit (429) | High | Microsoft Graph rate-limited the tenant/app/mailbox after the Graph SDK's `Retry-After` retries were exhausted. Surfaces as `Too many requests.`, `The app or user has been throttled.`, or `(HTTP Status Code: TooManyRequests) Batching request failed with an unknown reason.` Most likely under high call volume — `ForEachFileFolderConnections`, `ForEachEmailConnections`, `BulkAddListItemsConnections`, retrieve-all reads (`MaxResults = 0`), or batching. No resource to fix — resolved by backoff / lowering the call rate. (On package 3.9.11 the SDK "retries exhausted" case surfaces raw, not as `Too many requests.` — see the playbook.) | [request-throttled.md](./playbooks/request-throttled.md) |
| Authentication / Token Invalid (401) | Medium | Any Mail/Files/Excel activity faults at authentication: a raw token message (e.g. `Access token has expired or is not yet valid.`), `The caller is not authenticated.`, an `AADSTS<code>` configuration error (consent / credentials), an interactive-login timeout, `No default connection is available.`, or `Automation Cloud cannot be reached...`. Expired/invalid token, revoked/missing consent, cancelled interactive login, or missing/unauthenticated connection. | [authentication-token-invalid.md](./playbooks/authentication-token-invalid.md) |
| Transient Service Error / Timeout (5xx) | Medium | Server/transport-side failure, generally not user-correctable: `The server is unable to process the current request.` (503), `Request time out.`, a raw 500/504/timeout inner message, the Mail `... modify your Query or Top values ...` 500, a batched 5xx, or `Automation Cloud cannot be reached...`. Resolved by retry with backoff. Distinct from 429 throttling. | [transient-service-error.md](./playbooks/transient-service-error.md) |
| Mail — Message Not Found | High | Message-by-ID Mail operation (`GetEmailByIdConnections`, `ForwardEmailConnections`, `ReplyToEmailConnections`, `MarkAsReadUnreadConnections`, `DeleteEmailConnections`, `ArchiveEmailConnections`, `DownloadEmail*`, legacy `ForwardMail`/`ReplyToMail`/`DeleteMail`/`SetMailCategories`/`GetMail` with `EmailId`, `NewEmailReceived` by-ID fetch) cannot resolve the message. Surfaces as `The resource could not be found.`, raw `ErrorItemNotFound` (`The specified object was not found in the store.`), `ErrorInvalidMailboxItemId` (`Item ... doesn't belong to the targeted mailbox`), or `ErrorInvalidIdMalformed` (`Id is malformed.` — corrupted/truncated stored ID; Outlook IDs are integrity-checked). Message deleted/moved, mailbox-scoped ID used against another mailbox, or stale/corrupted persisted ID. | [mail-message-not-found.md](./playbooks/mail-message-not-found.md) |
| Mail — Invalid OData Query | High | Mail read (`GetMail`, `GetNewestEmail`, `GetEmailListConnections`, `ForEachEmailConnections`, filter-based triggers) faults with `Invalid Query. Please use OData format for filter queries. Press F1 for examples.` (fixed text, both legacy and Connections) or a raw Graph invalid-filter message. Malformed `$filter` syntax, wrong property names, or bad literal formats. Deterministic — fixed by correcting the query. | [mail-invalid-odata-query.md](./playbooks/mail-invalid-odata-query.md) |
| Mail — Send / Forward / Reply Rejected | Medium | Send-path fault on the message itself: `File does not exist: <path>` (missing attachment), or raw Graph rejections matched by code family — `ErrorInvalidRecipients`, `ErrorSendAsDenied` (shared mailbox without Exchange Send As), `ErrorMessageSizeExceeded`. `SendMailConnections`, legacy `SendMail`, `ForwardMail`/`ForwardEmailConnections`, `ReplyTo*`. | [send-mail-rejected.md](./playbooks/send-mail-rejected.md) |
| Email Trigger — Connection Event Failure | Medium | `NewEmailReceived`/`EmailSent` trigger faults at the Integration Service connection layer — raw `UiPath.ConnectionClient.Contracts.ConnectionHttpException` or `Office365Exception` carrying the identical message (HTTP status embedded) — while fetching the event email, debug sample, or connection token. Connection deleted/unauthorized/not in folder, or IS-side 5xx. | [email-trigger-connection-event-failure.md](./playbooks/email-trigger-connection-event-failure.md) |
| Legacy Mail — NullReferenceException | Medium | Raw `System.NullReferenceException` from legacy `SendMail`/`ForwardMail`/`ReplyToMail` — null `Message` input or null element in `To`/`Cc`/`Bcc` arrays. Raw NRE type always indicates a legacy activity (Connections path remaps it to `The object used in the activity does not exist.`). | [legacy-mail-null-reference.md](./playbooks/legacy-mail-null-reference.md) |
| Copy Item — Null DriveItem | High | Legacy `CopyItem` throws raw `ArgumentNullException: Value cannot be null. (Parameter 'DriveItem')` — the bound variable is null because an upstream `FindFilesAndFolders`/get-item matched nothing. Null parent is legal (OneDrive root fallback); only `DriveItem` faults. | [copy-item-argument-null.md](./playbooks/copy-item-argument-null.md) |
| Create Folder — Invalid Name or Path | High | `CreateFolderConnections` input errors: `Value cannot be null. (Parameter 'Folder name'/'Folder path'/'Parent folder')`, `Folder path must contain at least one segment.`, `Folder path segment '<segment>' cannot have leading or trailing whitespace.`, or `Cannot create folder path: '<segment>' is a file, not a folder.` | [create-folder-invalid-path.md](./playbooks/create-folder-invalid-path.md) |
| Download — Conversion or Destination Error | High | `DownloadFileConnections` / legacy `DownloadFile`/`ExportAsPdf`: `Folders cannot be downloaded with this activity. ...`, unsupported PDF source extension, non-`.pdf` destination (`Local File Path should point to a folder or directly to a .pdf file.`), empty filename, or shared-item parent unresolvable (`Cannot locate file. Please ensure that you have proper permissions to access it.`). | [download-file-conversion-or-destination.md](./playbooks/download-file-conversion-or-destination.md) |
| Upload — Quota, Size, or Session Failure | Medium | `UploadFilesConnections` / legacy `UploadFile`: `The user has reached their quota limit.` (storage full — NOT throttling), `Max file size exceeded.` / `Maximum stream size exceeded.`, or `Upload session failed/incomplete/not found.` (transient chunked-upload break). Raw Graph wording on legacy. | [upload-file-quota-or-size.md](./playbooks/upload-file-quota-or-size.md) |
| Application Scope / Connection Misconfiguration | Medium | Faults **before any Graph call**, from the Microsoft 365 Application Scope's own configuration: `Could not retrieve the selected asset: ...`, `You must provide a literal value for <field> ...`, `The authentication parameters could not be read`, `You must provide a value for <field>`, `Please select an account.`, or placement errors (`Activity is only valid inside an Office 365 Scope` / `Cannot be used outside of a MicrosoftOffice365 Application Scope or outside of a Use OneDrive/SharePoint activity`). Fixed in Studio / Orchestrator configuration. | [application-scope-misconfigured.md](./playbooks/application-scope-misconfigured.md) |
