# Excel Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for Excel Activities investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Delete Range Failures | Medium | `DeleteRange` (classic) / `DeleteRangeX` (modern) fails. Signatures: `ArgumentException: The range is invalid` / `The string argument cannot be empty`; `COMException` referencing the shift operation or `Application-defined or object-defined error` (`0x800A03EC`); `IOException` / `COMException` with "cannot access the file"; or no exception with wrong rows removed. Causes: activity outside its required scope (`Excel Application Scope` / `Use Excel File`), invalid or empty `Range` syntax, `ShiftCells`/`ShiftOption` conflict with merged cells / Excel Tables, workbook locked or read-only, or AutoFilter hiding rows in the target block. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, and Delete-Range-when-Clear-Range-was-meant | [delete-range-failures.md](./playbooks/delete-range-failures.md) |
| Use Excel File / Excel Application Card Failures | Medium | The Modern scope container `ExcelApplicationCard` (Use Excel File) or Classic `Excel Application Scope` fails to open the workbook or govern its lifecycle. Signatures: `BusinessException: Error opening workbook. Make sure Excel is installed.`; `ArgumentException: Excel File path is empty or not set` / `Illegal characters in path`; `COMException RPC_E_SERVER_UNAVAILABLE` (`0x800706BA`) / `RPC_E_DISCONNECTED` (`0x80010108`) / `CO_E_SERVER_EXEC_FAILURE` (`0x80080005`); `BusinessException` on a child activity about missing scope; `COMException` / `BusinessException` referencing sensitivity label or information protection. Causes: Excel not installed or Interop broken (Classic always needs COM; Modern needs COM when COM-forcing properties are set or `.xls`/`.xlsb` files are involved); empty / null / illegal `WorkbookPath` from variable scope drift or upstream producer returning empty; COM/RPC race across multiple scopes without an `Excel Process Scope` wrapper (or a child macro tore down EXCEL.EXE); Modern child activity dropped outside any card (cross-cut to per-activity playbooks); Microsoft Purview / AIP sensitivity-label rejection on pre-v2.23.4 packages or v2.23.4+ without the SensitivityLabel/Operation properties configured. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, Visible-True-to-debug (a COM-forcing footgun), and package-downgrade-to-silence-sensitivity-label | [excel-application-card-failures.md](./playbooks/excel-application-card-failures.md) |
| Execute Macro / Run Spreadsheet Macro Failures | Medium | VBA macro invoked via `ExecuteMacro` (classic) or `Run Spreadsheet Macro` (modern) fails — most commonly `System.Runtime.InteropServices.COMException` with HRESULTs `0x80020009 DISP_E_EXCEPTION`, `0x800AC472`, or `0x80010108 RPC_E_DISCONNECTED`; sometimes a job hang with no exception when Excel surfaces a modal dialog. Causes: macro name not in workbook, VBA error inside the macro, macro tears down Excel (Workbooks.Close / Application.Quit), modal dialog (MsgBox / InputBox / debug break) blocking COM, macros disabled by Trust Center policy on the Robot host, concurrent COM access (STA apartment violation), or missing add-in / ActiveX dependency. Includes anti-patterns section rejecting Delay-as-fix and bare-Try-Catch suppression | [execute-macro-failures.md](./playbooks/execute-macro-failures.md) |
