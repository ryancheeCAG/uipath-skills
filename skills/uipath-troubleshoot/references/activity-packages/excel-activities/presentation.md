# Excel Activities Presentation Rules

- **Activities** — use the display name in prose (e.g., "Read Range", "Write Range", "Use Excel File"). Include the class name in parentheses on first mention so an agent can match a stack trace (e.g., "Read Range (`ReadRangeX`)"). Don't lead with the fully qualified class name.
- **Scopes** — distinguish "Excel Application Scope" (legacy, COM-only, requires Excel installed, holds Excel UI open) from "Use Excel File" (modern, OpenXML by default, falls back to COM for unsupported features). When the scope matters for the diagnostic, name it explicitly.
- **Workbooks** — refer to a workbook by its filename with parent context (e.g., "workbook `Q1-Forecast.xlsx` in `C:\Robot\Data\`"), not by full path on first mention. Show the full path when the path itself is the subject of the error (file-not-found, UNC issues) or when the agent needs to inspect the host. Show casing exactly as the workflow configured it.
- **Sheets** — refer to a sheet by its name in single quotes (e.g., "sheet 'Sheet1'", "sheet 'Q1 Data'"). Sheet name comparison is case-sensitive on the OpenXML provider and case-insensitive under Excel COM — when reporting a sheet-name mismatch, show both the configured name and the actual sheet names in the workbook verbatim.
- **Ranges** — show ranges in A1 notation exactly as configured (e.g., `A1:D100`, `B2`, blank for used-range). Don't normalize, expand, or trim.
- **Providers** — call out "OpenXML provider" vs. "Excel COM" when the diagnostic depends on which one ran. The provider affects which exceptions are produced and which workbook features are supported.
- **Host processes** — when referencing Excel processes on the host, use the literal `EXCEL.EXE` (uppercase) — that's the name as it appears in Task Manager and `tasklist`. Distinguish a user-owned instance (Excel UI visible to a logged-in user) from an orphan (no UI, owned by the Robot's session or a prior job's session).
- **Local file paths** — show the full local path as configured on the activity. Distinguish local paths from network paths (`\\server\share\...`) explicitly — lock semantics, latency, and permission models differ.
