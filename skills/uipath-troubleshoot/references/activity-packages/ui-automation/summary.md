# UI Automation Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for UI Automation investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Selector Failure — Healing Fix Available | High | Selector mismatch with Healing Agent fix in `healing-fixes.json` | [selector-failure-healing-fix.md](./playbooks/selector-failure-healing-fix.md) |
| Selector Failure — Healing Disabled | High | Selector mismatch with Healing Agent not enabled on the process | [selector-failure-healing-disabled.md](./playbooks/selector-failure-healing-disabled.md) |
| Selector Failure — Manual Investigation | Medium | Selector mismatch requiring manual analysis (HA produced no fix or source code available) | [selector-failure-manual.md](./playbooks/selector-failure-manual.md) |
| Selector Failure — Scope Container Attached to Wrong Page/Window | Medium | Scope container (`NApplicationCard`, `NBrowser`, `Use Application/Browser`, `NWindow`) attached to a different page/window than the inner selector expects; the selector itself is correct for its intended page | [scope-container-wrong-page.md](./playbooks/scope-container-wrong-page.md) |
| Disabled Element — AlterIfDisabled Not Enabled | High | `UiNodeDisabledElementException` from any UIAutomationNext interaction activity (NClick, NTypeInto, NSetText, NCheck, NSelectItem, NSAPClickPictureOnScreen) where the target was found but disabled and `AlterIfDisabled` was not set | [disabled-element.md](./playbooks/disabled-element.md) |
| Verify Execution Failure — Post-Action Assertion Did Not Hold | High | `VerifyActivityExecutionException` from NClick/NHover/NKeyboardShortcuts/NTypeInto where the action succeeded but the `VerifyOptions` post-condition (Appears / Disappears / TextChanges / AspectChanges) did not hold. Covers wrong verify target, wrong mode, timeout too short, post-recovery verify failure, action-had-no-effect. NTypeInto text-mismatch sub-cases not included. | [verify-execution-failure.md](./playbooks/verify-execution-failure.md) |
| Click Coordinate Off-Screen — HardwareEvents | High | Input activity (`NClick`, `NTypeInto`, ...) faulted with `UiAutomationException: "Cannot send input to UI element because it is outside of screen bounds."` because `InteractionMode=HardwareEvents` requires an on-screen coordinate but the design-time-captured coordinate falls outside the runtime host's virtual screen. Element was located; coordinate rejected. | [click-coordinate-off-screen.md](./playbooks/click-coordinate-off-screen.md) |
| Ambiguous Selector — Target Matched Multiple Elements | High | `NodeAmbiguousException` from any UIA-Next activity that resolves a target. Selector matched MORE than one element — too generic, repeated UI pattern, multiple windows of the same app, dynamic-attribute drift, iframe / shadow-root double traversal, or a dialog hoisting a duplicate. Distinct from `NodeNotFoundException` (zero matches). Healing Agent explicitly bypasses this exception — no recovery data is produced. | [ambiguous-selector.md](./playbooks/ambiguous-selector.md) |
| Timeout Issue | Low | UI automation activity exceeded its timeout waiting for an element or application state | [timeout-issue.md](./playbooks/timeout-issue.md) |
| Healing Agent — Orchestrator Status, Logs & Notifications (Is This Expected?) | High | "Is this expected?" disambiguator for 4 Orchestrator-side HA surfaces, routed by where the customer saw it: (1) Jobs-grid `Issues Detected` / tab "Unlicensed version" banner, (2) Info log `Healing agent configuration.` (benign config read), (3) email/in-app Notification Summary (real HA event), (4) `No available license / Agentic units` informational notice. Covers preview UIA `24.10.x` detecting/alerting unlicensed + the shared "does this consume units?" rule. Real no-license error → healing-agent-no-license.md | [healing-agent-orch-issues.md](./playbooks/healing-agent-orch-issues.md) |
| Healing Agent — No License / Heals Available | High | Robot log contains `"No available license / Agentic units to perform healing analysis and recovery"` (or, corroborating: HA enabled + 22-byte healing-data ZIP + `Allowed.AgentService == 0`) | [healing-agent-no-license.md](./playbooks/healing-agent-no-license.md) |
| Healing Agent — No Recovery Data | Low | Healing Agent enabled but no recovery data generated after failure (license OK; connectivity / classic activity / image-only target) | [no-recovery-data.md](./playbooks/no-recovery-data.md) |
| Application Not Found | High | `ApplicationNotFoundException` thrown by `NApplicationCard` (Use Application / Use Browser) when target app can't be located and `OpenMode=Never`. Covers `OpenMode` misconfiguration, selector drift, crashed-app, no-visible-window, cold-start race. Distinct from `ApplicationOpenException` (launch attempted) and `WrongTargetApplicationException` (wrong window matched). | [application-not-found.md](./playbooks/application-not-found.md) |
| Application Could Not Open — Launch Failed | High | `ApplicationOpenException` ("Could not open target application.") — a `Use Application/Browser` scope with `Open` ≠ `Never` tried to launch the app and failed (missing executable, blocking startup dialog, browser/WebDriver, permissions) | [application-open-failed.md](./playbooks/application-open-failed.md) |
| Identified Element Belongs to a Different Application | High | `WrongTargetApplicationException` ("The identified element does not belong to the target application/browser.") — the element acted on is owned by a different process than the scope's target (foreground/focus change, loose application selector) | [wrong-target-application.md](./playbooks/wrong-target-application.md) |
| Go To URL — Navigation Failed | Medium | `Go To URL` (`NGoToUrl`) could not navigate: missing `URL`, `BrowserInvalidURLException`, `BrowserFailedToNavigateToUrlException`, or a local `file://` blocked on Chromium | [browser-navigation-failed.md](./playbooks/browser-navigation-failed.md) |
| Inject Js Script — Script Missing or Execution Failed | Medium | `Inject Js Script` (`NInjectJsScript`) with empty `Script code`, or the JavaScript threw / could not run in the browser | [inject-js-failed.md](./playbooks/inject-js-failed.md) |
| Activity Configuration Error — Invalid Property Value | High | An activity faulted on its own property validation: Mouse Scroll `Movement units` < 1 or missing searched element, Keyboard Shortcuts empty/invalid sequence, Take Screenshot missing `File name` / `Saved image` | [activity-configuration-error.md](./playbooks/activity-configuration-error.md) |
| Element Found But Not Actionable | Medium | Element located but stale (`InvalidNodeException` / `UiNodeUninitializedElementException`), not visible (no screen rectangle / `TargetFoundButNotVisibleException`), or blocked by a browser dialog (`TargetNotFoundBrowserBlockedException`) | [element-found-not-actionable.md](./playbooks/element-found-not-actionable.md) |
| Select Item — Container Has No Items | Medium | `UiNodeHasNoItemsException` ("The target container does not have any items.") — Select Item's dropdown/list was empty at action time | [select-item-no-items.md](./playbooks/select-item-no-items.md) |
