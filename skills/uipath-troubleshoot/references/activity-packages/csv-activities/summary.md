# CSV Activities Playbooks

**Overview:** [overview.md](./overview.md) ‚Äî CSV file activities (`Read CSV` / `Write CSV` / `Append To CSV`) in `UiPath.System.Activities`, the bundled `CsvHelper` dependency, and common failure patterns

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Append/Write CSV ‚Äî "Method not found: 'CsvHelper...'" | High | `CsvHelper` version conflict: the library is bundled in **both** `UiPath.System.Activities` and `UiPath.Excel.Activities`; mismatched package versions bind an incompatible `CsvHelper` and the activity throws `Method not found`. Fix: align both packages (upgrade to current stable) | [csv-helper-method-not-found.md](./playbooks/csv-helper-method-not-found.md) |
| Append/Write CSV ‚Äî file locked or invalid / cloud path | Medium | `The process cannot access the file because it is being used by another process` (CSV open in Excel / held by another job) or `The filename, directory name, or volume label syntax is incorrect` (a raw `https://` SharePoint/OneDrive path ‚Äî CSV activities are local-file only). Fix: close/kill the lock holder; for cloud, download local ‚Üí append ‚Üí upload via Microsoft 365 activities | [csv-file-locked-or-invalid-path.md](./playbooks/csv-file-locked-or-invalid-path.md) |
| Append/Write CSV ‚Äî DataTable structure / null mismatch | Medium | Mismatched columns or rows fail to write: an uninitialized `DataTable`, a column count/header mismatch vs the existing CSV, or mapping out-of-scope fields. Fix: instantiate the `DataTable` (Build Data Table with matching headers) before the Add Data Row loop; align column structure | [csv-datatable-structure-mismatch.md](./playbooks/csv-datatable-structure-mismatch.md) |
| Read CSV ‚Äî "Line X contains more values than the header line" | Medium | A data row has more fields than the header: the `Delimiter` property doesn't match the file (comma vs semicolon/tab), or a field contains the delimiter unquoted (e.g. `1,000`). Fix: set the correct `Delimiter`, use `IgnoreQuotes` for quoted fields, or Read Text File ‚Üí Generate Data Table for erratic data | [read-csv-more-values-than-header.md](./playbooks/read-csv-more-values-than-header.md) |
| Read CSV ‚Äî "The CSV file format for [Path] is invalid" | Medium | Leading blank lines at the top of the file, or a `Has headers` mismatch (checked when there's no title row, or vice versa), breaks the tabular structure. Fix: strip leading blank lines; set `Has headers` to match the file | [read-csv-invalid-format.md](./playbooks/read-csv-invalid-format.md) |
| Read CSV ‚Äî "Could not find file" / "Could not find a part of the path" | Medium | The path is broken or evaluated too early: an async download/export hasn't finished writing, the path variable is wrapped in quotes (literal `"MyPathVar"`), or a relative path resolves wrong. Fix: File Exists retry/Delay before the read; pass the variable unquoted; use a correct absolute path | [read-csv-file-not-found.md](./playbooks/read-csv-file-not-found.md) |
| Write CSV ‚Äî "Cannot set unknown member" | Medium | Activity-package version skew: the robot/runtime has a different (older) `UiPath.System.Activities`/`UiPath.Excel.Activities` than the build machine, so a property set in the XAML doesn't exist in the runtime activity. Works on dev, fails on the robot. Fix: align the runtime's package versions with `project.json` (or rebuild against the runtime's versions) | [write-csv-cannot-set-unknown-member.md](./playbooks/write-csv-cannot-set-unknown-member.md) |
| Write CSV ‚Äî "Failed to create a 'Delimitator' from the text '...'" | High | The `Delimiter` property is set from a string/localized name (`"Tab"`, `"Comma"`, `"Pipe"`) instead of the enum value or character literal. Fix: pick the delimiter from the dropdown, or use the character literal (`vbTab`, `","`, `"|"`) | [write-csv-invalid-delimiter.md](./playbooks/write-csv-invalid-delimiter.md) |
| Write CSV ‚Äî "Access to the path is denied" | Medium | `UnauthorizedAccessException`: the robot user lacks write permission, the file is read-only, or it is open in Excel. Fix: grant Read/Write to the folder, clear read-only, or close/Kill Process EXCEL before the write (vs the file-lock playbook's 'being used by another process') | [write-csv-access-denied.md](./playbooks/write-csv-access-denied.md) |
| Write CSV ‚Äî "Unsupported encoding name" | High | The `Encoding` property is set to a string .NET doesn't recognize (typo / non-canonical name). Fix: use a valid .NET encoding name (`UTF-8`, `Windows-1252`), leave it default, or use Output Data Table ‚Üí Write Text File | [write-csv-unsupported-encoding.md](./playbooks/write-csv-unsupported-encoding.md) |
