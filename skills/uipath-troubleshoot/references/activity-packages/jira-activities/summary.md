# Jira Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for Jira Activities investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Authentication information is invalid | High | `Jira Scope` faults at scope open with `Authentication information is invalid. Please check your credentials...`. Atlassian rejected the credential. Sub-cases: (A1) `Api Token` passed as a plain `String` instead of `SecureString`; (A2) `Username` is an alphanumeric `accountId` instead of the account email; (A3) leftover `Client Id` / `Client Secret` conflict with `Authentication Type = Api Token`; (A4) basic password auth used on an org that enforces MFA/SSO — Atlassian blocks passwords, an API token is required | [jira-scope-authentication-failures.md](./playbooks/jira-scope-authentication-failures.md) |
| Response was not recognized as JSON / HTTP 500 | High | A child Jira activity faults with `Response was not recognized as JSON` or an HTTP `500`. Structural / routing problem, not credentials. (R1) `Server URL` points past the root instance (ends in `/secure/Dashboard.jspa` or a project path) so the REST call hits HTML; (R2) the target is an on-premises **Server / Data Center** instance whose endpoints diverge from the Jira Cloud shape the classic pack expects | [jira-scope-response-not-json-or-500.md](./playbooks/jira-scope-response-not-json-or-500.md) |
| This activity is either missing or could not be loaded properly | Medium | A Jira activity will not load. The legacy pack's transitive **RestSharp** (or similar) version is overridden by another package in the project, so the activity assembly cannot bind — `This activity is either missing or could not be loaded properly` at design time, or `FileLoadException` / `TypeLoadException` at runtime. Resolve the version conflict or migrate to the Integration Service Jira connector | [jira-activity-missing-or-not-loaded.md](./playbooks/jira-activity-missing-or-not-loaded.md) |
| Transition Issue fails or is rejected | Medium | `Transition Issue` faults moving a ticket to a new status. (T1) a required field on the transition screen (`Resolution`, `Fix Version/s`, `Assignee`) was not supplied — `Field '<name>' is required` / `400`; (T2) a hardcoded transition ID is not legal from the issue's **current** status — `Transition '<id>' is not valid ...` (use `Get Transitions` and resolve the `.Id` dynamically); (T3) a workflow Condition/Validator or a missing project permission blocks the robot account (confirm by transitioning manually as the service account); (T4) `Error converting value '...' to type 'Atlassian.Jira.IssueFieldEditMetadataOperation'` — a JSON metadata-parsing bug in older pack versions (upgrade, or migrate to the Integration Service Jira connector) | [jira-transition-issue-failures.md](./playbooks/jira-transition-issue-failures.md) |
