# Intelligent OCR / Document Understanding (Classic) Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — data correlation rules and testing prerequisites for Intelligent OCR / Document Understanding Activities investigations

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Document Understanding — License or Endpoint Rejected the Call | Medium | A DU activity's HTTP call to the Document Understanding server is rejected with `UiPath.SmartData.Utils.DocumentUnderstandingClient.DUApiException` (carries HTTP status, `CF-RAY`, `AppId`): `Your license could not be validated...` (401), `Failed to consume the requested number of pages...` (403), `You have exceeded the request size limitations...` (413), `The service has rejected the request. Please make sure you are using the correct endpoint...` (400), `Response signature is invalid. Endpoint is not supported.`, or generic `DocumentUnderstanding server returned <code> ...` (5xx). | [du-license-or-endpoint-rejected.md](./playbooks/du-license-or-endpoint-rejected.md) |
| Document Understanding — Not Enabled on Tenant / Tenant Key | Medium | A DU activity faults resolving the tenant context before processing: `Failed to fetch Document Understanding projects list. Please connect to a tenant that has Document Understanding enabled.`, `Couldn't retrieve a tenant key.`, or `Information about tenant <id> couldn't be retrieved.`. DU isn't enabled/licensed on the connected tenant, or the robot is on the wrong tenant. | [du-not-enabled-or-tenant-key.md](./playbooks/du-not-enabled-or-tenant-key.md) |
| Document Understanding — Storage Bucket / Taxonomy / Folder Missing | Medium | A DU activity can't read/write a storage location: `No such bucket named '<name>'`, `Couldn't find path <p> in bucket <b> associated with '<x>'`, `Could not load the <x> from storage bucket <b> and path <p>`, `The local directory path '<path>' could not be found.`, or `Could not find Orchestrator Folder '<name>'`. The taxonomy/model/results blob, bucket, folder, or local directory doesn't exist where the activity looked or isn't accessible to the robot. | [du-storage-or-taxonomy-missing.md](./playbooks/du-storage-or-taxonomy-missing.md) |
| Document Understanding — Document Rejected at Validation Station | Medium | `Present Validation Station` faults with `UiPath.IntelligentOCR.Exceptions.DocumentRejectedByUserException` because a human reviewer rejected the document (carries `DocumentRejectionReason` + `Attributes`). A business outcome surfaced as an exception — handle the rejection path in the workflow, not a technical repair. | [du-document-rejected.md](./playbooks/du-document-rejected.md) |
