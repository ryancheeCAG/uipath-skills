# Testing Activities Playbooks

**Investigation guide:** [investigation_guide.md](./investigation_guide.md) — execution-context rules (Test Job vs Studio), distinguishing a designed test failure from an execution fault, and testing prerequisites for `UiPath.Testing.Activities` investigations.

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Attach Document — File Not Found | High | `AttachDocument` (`UiPath.Testing.Activities`) faults with `System.IO.FileNotFoundException` (message = the file path) because `File.Exists(FilePath)` is checked before any Orchestrator call. Causes: wrong path, relative-path vs robot working-directory mismatch, file not produced by an upstream step, dev-vs-robot machine path, unreachable UNC share, or file deleted between steps. | [attach-document-file-not-found.md](./playbooks/attach-document-file-not-found.md) |
| Compare Text — Output Report Write Failure | High | `CompareText` faults with `System.UnauthorizedAccessException` writing its HTML diff to `OutputFilePath` (default `differences.html`): protected/system directory, ACL-restricted folder, path is a directory, file locked, or the relative default resolving to a non-writable location on an unattended robot. Distinct from the designed `TestingActivitiesException: The analyzed texts are different.` | [compare-text-output-write-failures.md](./playbooks/compare-text-output-write-failures.md) |
| Verify Expression — Failures | Medium | `VerifyExpression`: either a **designed** assertion fail (`TestingActivitiesException: Verification failed. The expression '<expr>' returned 'False'.` with `ContinueOnFailure = false`) or a `System.NullReferenceException` thrown while evaluating the user's `Expression` operand (null dereference / uninitialized variable) before the assertion. | [verify-expression-failures.md](./playbooks/verify-expression-failures.md) |
| Verify Expression With Operator — Failures | Medium | `VerifyExpressionWithOperator`: three surfaces — designed assertion fail (`TestingActivitiesException: Verification failed. The expression '<first>'<v1> was not <op> the expression '<second>'<v2>.`), operand `NullReferenceException`, or `System.Net.Http.HttpRequestException` when the assertion-result POST to Orchestrator/Test Manager fails (Test Job only; verification logic is not the cause). | [verify-expression-with-operator-failures.md](./playbooks/verify-expression-with-operator-failures.md) |
| Get Test Data Queue Item — Failures | Medium | `GetTestDataQueueItem`: clear `TestingActivitiesException` messages (`Queue is empty or all items are consumed`, `Queue <name> not exist.`, content-null, `Test data queue item is not a JSON: <error>`) versus a bare `System.NullReferenceException` from an unbound `QueueName`/`Output` or a null dictionary field dereferenced downstream. | [get-test-data-queue-item-failures.md](./playbooks/get-test-data-queue-item-failures.md) |
| Coverage Merge — Failures | Low | Internal `CoverageMergeActivity` (`[Browsable(false)]`, runs after a coverage-enabled test set) faults with `System.InvalidOperationException` (`Sequence contains no elements` on empty/corrupt coverage) or `InvalidArgumentsException` (null `TestSetExecutionId`/`PackageId`). No user-facing config — re-run, disable coverage to unblock, escalate. | [coverage-merge-failures.md](./playbooks/coverage-merge-failures.md) |
