## 1. Process Overview

| Field | Value |
|-------|-------|
| Full name | Vendor Payment Sync |
| Department | Finance — Accounts Payable |
| Description | Pull approved vendor payments from the ERP REST API, validate each against the vendor master in the SQL database, post confirmed payments back to the ERP, and email a daily reconciliation summary. Runs unattended; no UI automation. |
| Required roles | Unattended Robot |
| Schedule | Daily at 6 AM, Monday–Friday |
| Volume | 200–400 payments per day |
| Input | Pending-payment batch from the ERP REST API |
| Output | Posted payments in the ERP; reconciliation email |

### In Scope
- Read pending payments from the ERP REST API
- Validate each payment against the vendor master (SQL database)
- Post confirmed payments back via the ERP REST API
- Email a reconciliation summary via SMTP

### Out of Scope
- Vendor onboarding
- Manual exception remediation

## 17. Testing Strategy

Happy path (valid payments post and reconcile), edge cases (empty batch, partial-day batch), and error scenarios (API timeout, DB unreachable, SMTP failure) are all in scope. Test cases exercise `ValidatePayment.xaml` rules and the REFramework retry/transaction flow.
