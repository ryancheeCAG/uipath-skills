Deploy Commands — Real Examples

Exact command calls with real values filled in

Dashboard being deployed
APP_NAME"Agent Health Dashboard"from state.json — used as-is in -n flag
ROUTING_NAME"agent-health-x7k2"permanent URL slug
SEMVER"1.0.0"current version
NEXT_SEMVER"1.0.1"auto-bumped
FOLDER_NAME"Shared"auto-resolved
FOLDER_KEY"a1b2c3d4-e5f6-7890-abcd-ef1234567890"from folder list
Scenario A First-ever deploy — user chooses to pin to Governance UI
Pre-flight
$ uip login status --output json { "Result": "Success", "Data": { "Status": "Logged in", "Organization": "appsdev", "Tenant": "appsdevDefault" } }
Folder check — "Shared" found automatically
$ uip or folders list --all --output json > /tmp/uip-folders.json # Check for "Shared" folder… { "key": "a1b2c3d4-e5f6-7890-abcd-ef1234567890", "name": "Shared" } # → FOLDER_KEY and FOLDER_NAME set automatically. No user prompt.
Deploy plan shown to user
Your Agent Health Dashboard is ready to be deployed. 📦 Version: 1.0.0 → 1.0.1 🔗 URL path: agent-health-x7k2 📁 Folder: Shared 🔄 Type: Fresh deploy 📌 Do you want to pin this dashboard to the Governance UI? → Say "deploy and pin" to make it visible in the Governance section → Say "deploy" to deploy without pinning
User replies: "deploy and pin" → PIN_TO_GOVERNANCE=true
Step 5 — Production build
$ mv .env.local .env.local.deploy-bak $ npm run build ✓ built in 4.2s — dist/ ready (143 kB) $ mv .env.local.deploy-bak .env.local
Step 6 — Pack
$ uip codedapp pack dist \ -n "agent-health-x7k2" \ -v "1.0.1" \ --output json { "packagePath": "./agent-health-x7k2.1.0.1.nupkg", "name": "agent-health-x7k2", "version": "1.0.1" }
Note: -n takes the routing slug, not the display name
Step 7 — Publish
$ uip codedapp publish \ -n "agent-health-x7k2" \ -v "1.0.1" \ --output json { "DeploymentVersion": 3, "name": "agent-health-x7k2", "version": "1.0.1", "status": "Published" }
Step 8 — Deploy (pinned to Governance UI)
$ uip codedapp deploy \ -n "Agent Health Dashboard" \ --path-name "agent-health-x7k2" \ --folder-key "a1b2c3d4-e5f6-7890-abcd-ef1234567890" \ --tags "governance,dashboard" \ --output json { "SystemName": "app-7f3a", "AppUrl": "https://cloud.uipath.com/appsdev/appsdevDefault/apps_/app/agent-health-x7k2", "status": "Deployed" }
-n = sanitized display name (package lookup)
--path-name = routing slug → becomes the URL
--tags "governance,dashboard" → visible in Governance UI
Success message
🎉 Agent Health Dashboard is live. https://cloud.uipath.com/appsdev/appsdevDefault/apps_/app/agent-health-x7k2 Version 1.0.1 · Folder: Shared Your dashboard is now visible in the Governance section. To update it after changes, say "deploy this dashboard" again.
Scenario B Upgrade (second deploy) — no Governance pin — no "Shared" folder in tenant
state.json already has deployment info from Scenario A
{ "app": { "semver": "1.0.1", "routingName": "agent-health-x7k2" }, "deployment": { "systemName": "app-7f3a", ← non-null → this is an UPGRADE "folderKey": "a1b2c3d4-...", "folderName": "Shared", "pinnedToGovernance": true } }
Because systemName is set, folder resolution is skipped — already known.
Version bump
# SEMVER="1.0.1" → NEXT_SEMVER="1.0.2" $ uip codedapp list --output json # 1.0.2 not found → proceed
Deploy plan shown to user
Your Agent Health Dashboard is ready to be deployed. 📦 Version: 1.0.1 → 1.0.2 🔗 URL path: agent-health-x7k2 📁 Folder: Shared 🔄 Type: Updating existing deployment (app-7f3a) 📌 Do you want to pin this dashboard to the Governance UI? → Say "deploy and pin" to make it visible in the Governance section → Say "deploy" to deploy without pinning
User replies: "just deploy" → PIN_TO_GOVERNANCE=false
Steps 5–7 identical — build, pack, publish with new version
$ npm run build # (env.local swapped out and back) $ uip codedapp pack dist -n "agent-health-x7k2" -v "1.0.2" --output json $ uip codedapp publish -n "agent-health-x7k2" -v "1.0.2" --output json { "DeploymentVersion": 4, "version": "1.0.2", "status": "Published" }
Step 8 — Deploy (no Governance pin)
$ uip codedapp deploy \ -n "Agent Health Dashboard" \ --path-name "agent-health-x7k2" \ --folder-key "a1b2c3d4-e5f6-7890-abcd-ef1234567890" \ --tags "governance" \ --output json { "SystemName": "app-7f3a", "AppUrl": "https://cloud.uipath.com/appsdev/appsdevDefault/apps_/app/agent-health-x7k2", "status": "Deployed" }
--tags "governance" only — no dashboard tag because user didn't opt in
Edge case — "Shared" folder does not exist
