TIER 1 — CATALOG Exact alias match in capability registry → pure template substitution
Resolution algorithm
metric.name → alias table → capability entry
capability entry → template key (e.g. "line-chart")
capability entry → API namespace + method
capability entry → default chartType, deltaDir, icon
build script: load template file
→ replace all <PLACEHOLDER> markers
→ inject time constants
→ write widget file atomically
What the agent provides
Only the metric name + time range.

Build script derives everything else from the capability registry:
template, API call, chart type, axis keys, delta direction, icon name, title text, description.
TIER 2 — PARAMETRIC Known SDK service + custom filter/transform → typed descriptor → build script compiles to TS
Typed descriptor schema (agent fills values)
{
 "service": "queues",  // SDK module
 "method": "getAll",
 "filter": {
  "field": "failureRate",
  "op": "gt",        // gt|lt|eq|gte|lte
  "value": 0.2
 },
 "sort": { "field": "failureRate", "dir": "desc" },
 "displayAs": "ranked-table",
 "title": "High-Failure Queues",
 "columns": ["name","failureRate","total"]
}
Build script compiles descriptor → safe TS
Descriptor is validated by JSON schema before compilation.

Build script generates the SDK hook, filter expression, and column definitions from the descriptor fields — no arbitrary code paths.

Agent provides values, not code. Hallucination risk: values only (field names, threshold numbers).
TIER 3 — SANDBOXED CUSTOM Novel query → agent writes typed async fn body only → shell is always deterministic
Constrained interface (agent must satisfy)
// This interface never changes
type Row = Record<string, unknown>
type DataFn = (
  sdk: UiPathClient,
  getToken: () => Promise<string>
) => Promise<Row[]>

// Agent writes ONLY this body:
const r = await sdk.queues
  .getAll({ state: 'Faulted' })
return r.items?.map(q => ({
  name: q.name,
  pending: q.pendingCount
})) ?? []
Retry protocol on tsc failure
1. Build script injects fn body into typed template
2. Runs tsc --noEmit on that file only
3. If error: emits T3_RETRY:{"errors":[...]} to stdout
4. Agent reads error, rewrites fn body
5. Repeat max 3 times
6. If 3 failures: report to user with reason

Component shell (card, loading state, error boundary) is generated by build script — agent never writes JSX.

Hard Refuse — before any tier routing

Checked first, before classification. Agent refuses immediately with a plain-English reason + alternative suggestion.

• Agent cost in dollars (platform tracks AGU, not $)
• Real-time CPU/memory per agent (not exposed)
• Per-user job attribution (no end-user identity in records)
• Cross-tenant comparison (dashboards are single-tenant)
• SLA breach % (raw counts only, no SLA metadata)
• Per-agent memory profiling (fleet-level only)
• Historical queue throughput (no endpoint)
• Error message text / stack traces (no aggregation)
