Who is the primary user of this dashboard generation skill?

Their background determines how much we can abstract away — and how fast generation needs to feel

A

UiPath Admin / CoE Lead

Manages the automation platform. Understands Orchestrator, tenants, folders. Wants operational visibility — agent health, job failure rates, resource consumption — without writing code. Runs this once a week to set up a new dashboard for their team.

B

Business Analyst / Process Owner

Owns the automation use cases but has no platform knowledge. Doesn't know what Insights or Orchestrator are. Wants a dashboard that answers their specific question ("how many invoices did we process this month?") — nothing more, nothing less.

C

RPA Developer / Automation Engineer

Builds automations in Studio. Technically capable. Wants dashboards fast — doesn't want to write React or call APIs manually. Happy to understand what's happening under the hood.

D

All three — the skill serves all personas

The skill adapts its level of explanation and confirmation steps to whoever is asking. Technical users get a faster path; non-technical users get more guidance.
