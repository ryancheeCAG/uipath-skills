The answer: Tiered Resolution

Approach B doesn't refuse unknown metrics — it routes them through the right generation strategy. Three tiers, each progressively more flexible but equally safe.

TIER 1 — CATALOG

Known metrics

Exact match in capability registry. Fully deterministic.

"agent errors" → agents.getErrors
"invocation volume" → agents.getSummaryV2
"job failures" → jobs.getTopFailures
Generator: Pure template substitution
AI writes: Nothing
Speed: <1s per widget
TIER 2 — PARAMETRIC

Variations of known patterns

Known API + custom filter/threshold/group. Agent fills a typed parameter object. Build script compiles it.

"queues where failure > 20%"
→ sdk.queues.getAll
+ filter: failureRate > 0.2

"agents running > 5 min"
→ sdk.jobs.getAll
+ filter: duration > 300s
Generator: Typed query descriptor → TS
AI writes: Parameter values only
Speed: <2s per widget
TIER 3 — SANDBOXED CUSTOM

Truly novel queries

Agent writes a single hook function body — NOT a React component. Typed interface enforced by tsc. Component shell is always deterministic.

// Interface (never changes)
type DataFn = () =>
Promise<Row[]>

// Agent writes only this:
const r = await sdk
.queues.getAll({...})
return r.items ?? []
Generator: AI fn body + TS validation
AI writes: Fetch logic only, no JSX
Speed: 5–10s + tsc retry if needed

Resolution flow — every metric goes through this

User: "show me queues where failure rate > 20% and agent errors last 30 days"
↓ Agent extracts metrics: ["queue-failure-threshold:0.2", "agent-errors"]
↓ For each metric, build script runs resolution:
• "agent-errors" → T1 exact match → template substitution → ✅ done in 1s
• "queue-failure-threshold:0.2" → T1 miss → T2 parametric match on queues.getAll → typed descriptor → ✅ done in 2s
↓ Both widgets generated in parallel
↓ Progressive: first widget visible at 1s, second at 2s
↓ Dashboard preview open — total: ~5s after npm pre-warm

Key insight: the surface area AI can touch shrinks at each tier

Tier 1: AI touches nothing — pure substitution.
Tier 2: AI provides values in a typed JSON schema — build script validates before compiling.
Tier 3: AI writes one function body — typed interface enforced, component shell always deterministic, tsc --noEmit catches errors before the user ever sees output.

No unknown metric is refused. Every metric finds the safest path available.
