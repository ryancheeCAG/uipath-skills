LAYER 1
AGENT (NLP)
Intent Extractor
NLP → intent.json
metric names + time range + format hints
→
Plan Renderer
Plain-English plan from resolution results
No API names. One confirmation.
→
Tier 3 Hook Author
Only invoked for truly novel queries
Writes fetch fn body, not React
↓ intent.json + (optional) tier3 fn bodies
LAYER 2
RESOLUTION
Metric Classifier
alias lookup → T1 / T2 / T3 / refuse
→
Tier Router

T2: descriptor schema

→
Widget Config Builder

LAYER 3

UX SHELL

Dev Server

Current design — what moves out of the agent

  • Agent derives full TypeScript widget config → moved to Tier Router
  • Agent writes widget files → moved to Code Generator
  • Agent manages 7 sequential phases → moved to Build Engine pipeline
  • Agent guesses pre-warm timing → Pre-warm Manager guarantees it
  • Agent streams no progress → Progressive Emitter streams all events

New design — what the agent keeps

  • NLP → metric name + time range + format hints
  • Render plain-English plan from resolution output
  • Tier 3 only: write a typed async function body
  • Follow-up enhancements: classify "add / remove / change"
  • Refuse with reason when metric is in hard-refuse table
