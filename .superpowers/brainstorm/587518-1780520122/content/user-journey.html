Section 2: User Journey

What the user sees from first message to live dashboard — and every follow-up after that

Happy Path — First Dashboard

T+0s
User sends natural language request
"show me agent error rates, invocation volume, and queues where failure rate exceeds 20% for the last 30 days"
T+1s
Agent: intent extraction + pre-warm fires silently
Writes intent.json (3 fields). Starts npm ci in background — user never sees this.
Resolution Engine classifies all 3 metrics in parallel: T1, T1, T2.
T+3s
Agent shows plain-English plan (no API names)
Here's your Operations Health Dashboard — 3 widgets. Confirm to build, or tell me what to change.

• Agent Error Rate (30 days) — daily error counts as a trend line
• Invocation Volume (30 days) — total runs per day as an area chart
• High-Failure Queues — queues where failure rate > 20%, ranked table

What you can do: "make it 7 days", "add a KPI for total errors", "remove the queue widget", "add job failures"
T+20s
User reads plan, npm ci finishes in background
Pre-warm is guaranteed complete by the time user confirms — not timing-dependent. Build script polls for lock file.
T+25s
User confirms → Build Engine starts, streams progress
Building your dashboard...
✓ Agent Error Rate ready (1/3)
✓ Invocation Volume ready (2/3)
✓ High-Failure Queues ready (3/3)
✓ TypeScript check passed
✓ Dev server running
T+45s
Dashboard opens in browser
Agent shows: "Your dashboard is live at http://localhost:5173. Tell me what to change."

Follow-up Enhancements — Same Session

ADD WIDGET
"add a KPI showing total errors today"
1. Classify: ADD + metric resolution
2. Generate single widget file
3. Inject into Dashboard.tsx
4. tsc validate
5. Hot-reload fires automatically
~5s
MODIFY WIDGET
"change the error chart to 7 days"
1. Classify: CHANGE + target widget
2. Re-run resolution with new params
3. Regenerate widget file only
4. tsc validate
5. Hot-reload fires automatically
~5s
REMOVE WIDGET
"remove the queue widget"
1. Classify: REMOVE + target name
2. Delete widget file
3. Remove from Dashboard.tsx + index.ts
4. Update state.json
5. Hot-reload fires automatically
~2s

New session — resuming an existing dashboard

Agent detects .dashboard/state.json → reads widget list from state → starts dev server → ready for enhancements.
No rebuild needed. No re-asking for context. User continues where they left off.
