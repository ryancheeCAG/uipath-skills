How fast does "first dashboard ready" need to feel?

From the moment the user sends their request to a live, running dashboard in the browser

A

Under 30 seconds

Feels instant. The user barely has time to read the plan before the build is done. Requires aggressive pre-warming and parallelism. Some constraints on dashboard complexity.

B

30–90 seconds

Fast enough to hold attention. User reads the plan (~15s), approves, build completes while they're still engaged. This is the current design target. Achievable with current pre-warm approach.

C

90 seconds – 3 minutes

Acceptable if the experience is smooth and there's visible progress. User sees what's happening at each step. More time means richer dashboards (more widgets, better defaults).

D

It depends on complexity

A 3-widget dashboard should take under 60s. A 10-widget dashboard with custom queries can take 3–4 minutes. Progressive disclosure: show partial results as they're ready rather than blocking on full completion.
